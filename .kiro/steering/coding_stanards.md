# Coding Standards

* Use pyenv and use Python 311.1 which has been added to pyenv.
* Always use Object-Oriented Programming.
* Alwayse use snake case for file, methods, and variable names.
* Always use strict typing in Python.
* Follow PEP8 programmaing standards.
* Each import should be grouped together if they are from the same library and each group should be separated by one line. Imports should be ordered by line length.
* Always use try-except block for code that may throw exceptions.
* Create and seperate python files into packages if necessary to keep code tidy.
* No testing required because this is a lightweight application.
